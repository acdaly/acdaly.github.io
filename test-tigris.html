The Challenge

As a product design intern at LearnLab, I was tasked to redesign the workflow page of LearnSphere’s Tigris web application to improve usability.

LearnSphere is a project creating a shared repository of educational data and analysis tools. The Tigris web application is part of this software infrastructure— it enables researchers and professors alike to improve courses by analyzing educational data.

Scope

I had 8 weeks* to redesign Tigris’s workflow page while working on another LearnLab project.

As the sole designer on the project, I worked with Cindy Tipper (Project Manager and Developer) and 3 other developers.


*During my 8-week internship, I balanced simultaneously working on another LearnLab project. This project can also be seen on my portfolio as Personalized Learning².

Original Tigris workflow page

Solution

After completing 6 interviews with new and power users, I redesigned several parts of the workflow page.

My workflow page redesign

Improving Folder Hierarchy

New users often don’t realize that the components in the sidebar are draggable, and furthermore have a hard time understanding the folder hierarchy. I improved the hierarchy of text, contrast, and icons to make this more intuitive.

Increasing Canvas Space

Power users complained about not having enough canvas space, so I removed the top most banner, made the side menu and top bar thinner, and moved the zoom tool to the side.

Simplifying Components

The original component had 5 icons within it and often confused new users. I removed the “x” icon (it can now be deleted with the delete key or with a right click), changed the results icon from the magnifying glass to the documents icon, and made the name editable with a simple click.

Organizing the Banner

I removed the top most banner, made the side menu and top bar thinner, and moved the zoom tool to the side.

reflection

Communicating Design Decisions & Redesigning a Product

Through this project I learned how to communicate my critiques of a design, my user research findings, and my ultimate redesign. Because I was redesigning an existing product, it was critical that I was very clear about why I decided to make changes, especially when talking to non-designers leading the project.
